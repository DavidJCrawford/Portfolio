# Summary

In an agentic world, the quality of an agent's output is bounded by the quality of the information fed into it. Knowledge management — structured, accurate, machine-readable context — is therefore the primary [leverage point](./leverage-points.md), not the agents or the tools. An agent given rich, well-structured context produces dramatically better work than the same agent given poor or missing context.

# A working rule

Fill the gaps you *don't* want the model to hallucinate from — supply the facts and constraints that must hold. Leave the gaps where you *do* want it to generate — that's where the model's value lives. This very knowledge base is an instance of the idea: a portable, public context source (an [Open Knowledge Format](https://github.com/GoogleCloudPlatform/knowledge-catalog/tree/main/okf) bundle) so both people and agents can understand the project.

# Core idea

The leverage is the knowledge you give the system, not the system itself.
