# Summary

Building applications with readily-available foundation models, as distinct from traditional ML engineering that trains models from scratch ([Chip Huyen](../people/chip-huyen.md), *AI Engineering*). A central practice is **Evaluation-Driven Development**: define how you will evaluate the AI application *before* you build it, which forces clarity on what “good” even means. It's the disciplined counterpart to the craft of [product](./product.md) when the material is a probabilistic model.

# Core idea

Decide how you’ll measure success before you build — especially when the system is probabilistic.
