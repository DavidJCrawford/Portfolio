# Summary

Publish this brain (an [Open Knowledge Format](https://github.com/GoogleCloudPlatform/knowledge-catalog/tree/main/okf) bundle) in the public site repository, so people and agents can read it and understand how the project thinks — an applied example of [knowledge as leverage](../concepts/knowledge-as-leverage.md).

# Decision

Include the ideas, frameworks and positioning. Exclude all third-party personal information (only [David J Crawford](../people/david-j-crawford.md) is named; well-known authors may be cited as references). Keep employer product-roadmap specifics out and the industry context general.

# Rationale

A public, portable context source is more useful than private notes, and it walks the talk: the site argues that knowledge is the leverage point, so the project's own knowledge is made legible.
